To Framework or Not to Framework

February 1, 2015

In the last week of Phase 0 at DevBootcamp, I was given the opportunity to take another look at my blog site and do some revamping.

I was given the option of using a framework or using standard html and css.

Rather than do a complete overhaul in a framework, I opted to stay with html and css for my site and focus on some subtle refinements.

I chose this path for several reasons.

I think frameworks - like Bootstrap - are important and they are certainly ubiquitous. They take care of a lot of the work in terms of making a cohesive set of CSS styles and optimizations that are visually pleasing, responsive and translate well to multiple devices.

So why didnâ€™t I use a framework?

My goal in attending DBC was to learn the how and the why of all aspects of web development. I felt that using a framework at this point would allow me to skip over some important information.

I want to understand the underpinnings of advanced CSS techniques and how to translate them to mobile devices. I will be continually refining and reworking those aspects of my site.

have no doubt that I will use and implement frameworks in the future, both on this blog and on other sites and web applications. I look forward to embracing this technology along with the ability to create a true blog using Rails.
